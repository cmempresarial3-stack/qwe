Histórico e progresso do projeto

O app foi originalmente feito em React Native + Expo, usando uma versão antiga do SDK (51).

Houve um crash crítico relacionado ao bible-acf.json: o arquivo era um array, mas o código esperava um objeto.

Solução parcial: função loadBibleData() agora transforma automaticamente array → objeto + cache singleton.

Problemas de TypeScript foram corrigidos:

De 68 erros para 0 erros com configuração adequada do tsconfig.json.

Contextos Async (NotificationContext, etc.) foram protegidos com try/catch para evitar crash.

Carregamento de dados pesado (Bíblia 3.9MB) foi tornado async, para não bloquear a thread principal.

Testes finais mostraram que a Bíblia carrega corretamente, apenas quando a tela é aberta.

Problemas atuais

Apesar das correções, o app ainda não funciona corretamente:

Ao abrir o app, a splash screen aparece e carrega 100%, mas logo aparece uma tela de erro e o app fecha sozinho.

Códigos HTML ou cores (#A78BFA) aparecem na parte superior e inferior da tela – devem ser ocultos.

Há falhas contínuas de inicialização e crash ao carregar módulos JS (erros com RCTEventEmitter.receiveTouches() e problemas de registro de módulos).

Problemas de compatibilidade com Expo Go e versões de SDK (51 vs 54).

Ao tentar rodar o app pelo QR code no Expo Go, aparece a mensagem:

Something went wrong. Sorry about that.
You can go back to Expo home or try to reload the project.


Ao clicar em “carregar novamente” várias vezes, a tela fica branca e apenas carregando, nada aparece.

Necessidade de garantir que todas as dependências e plugins Babel, TypeScript, Expo, React Native e módulos nativos estejam corretos e funcionando.
